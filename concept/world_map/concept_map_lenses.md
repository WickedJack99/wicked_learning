# Map Lenses

## Overview

Map lenses are tools that change how the learner sees the world map.

Instead of implementing map views as ordinary UI dropdowns, lenses should be treated as world tools.

A learner may equip different lenses to reveal different layers of meaning.

## Purpose

Lenses allow the same map to show different information depending on the learner's current goal.

Examples:

- Exploration lens
- Competency lens
- Activity lens
- Hidden object lens
- Recommendation lens
- Social lens
- Event lens

## Lens as Tool

A lens may be represented as a tool in the learner companion's tool belt.

Example:

```text
Knowledge Lens
````

The learner activates a lens and the map changes visually.

## Lens Effects

A lens may affect:

```text
node visibility
node color
node glow
connection visuals
hidden objects
recommendations
competency indicators
activity indicators
particle effects
overlays
```

## Example Lenses

### Exploration Lens

Default view.

Shows the normal world appearance.

### Competency Lens

Highlights nodes based on competency state.

Examples:

```text
strong competency
needs refreshment
unknown topic
recently practiced
```

### Activity Lens

Shows what type of activity is currently available.

Examples:

```text
lesson
challenge
tool interaction
boss
merchant
portal
```

### Hidden Object Lens

Reveals hidden nodes, invisible doors or secret paths if the learner has unlocked the lens.

### Recommendation Lens

Highlights suggested paths.

## Custom Visuals

Lens visuals should be fully configurable.

A nature world might show:

```text
leaf-vein overlay
trail outlines
drifting pollen particles
```

A medieval world might show:

```text
magic glow
runes
fog clearing
```

## Design Principle

Lenses should make data feel like discovery.

The learner should not feel like they are switching dashboard filters.

They should feel like they are using a tool to perceive the world differently.
