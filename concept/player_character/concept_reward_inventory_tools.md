# Rewards, Inventory and Exploration Tools

## Design Philosophy

Rewards should not primarily function as external incentives.

Instead, rewards should serve one or more of the following purposes:

* Represent learner growth
* Unlock new exploration opportunities
* Provide new ways to interact with the world
* Create curiosity and discovery
* Visualize competency development

The goal is not to reward completion with arbitrary points but to make progress tangible within the learning world.

---

# Inventory System

Learners have access to a personal inventory.

The inventory is represented as a semi-transparent leather-style adventure journal or equipment pouch.

The inventory can be opened from anywhere in the learning world.

Items are displayed in a grid of inventory slots.

The inventory supports drag-and-drop interactions.

Items may be dragged onto interactive world elements to trigger actions.

Examples:

* Insert a key stone into a gateway
* Use a tool on a blocked path
* Activate a hidden mechanism
* Reveal secret content
* Unlock optional learning areas

The inventory should feel like an exploration tool rather than a reward screen.

---

# Knowledge Artifacts

Knowledge artifacts represent mastery of concepts, competencies or learning milestones.

Examples:

* Fleet Keystone
* Incident Response Seal
* Detection Engineering Crystal
* ECE Architect Fragment

Artifacts may be collected, combined or inserted into special world objects.

Example:

```text
Ancient Gateway

Required:
✓ Fleet Keystone
✓ ECE Keystone
✗ Detection Engineering Keystone
```

When all required artifacts are present, the gateway unlocks.

Artifacts serve as visual representations of learner progress.

---

# Exploration Tools

Some rewards unlock permanent world interaction capabilities.

These rewards function as tools rather than collectibles.

Examples:

* Climbing Boots
* Grappling Hook
* Navigation Compass
* Knowledge Lens
* Engineering Toolkit
* Translation Device

Tools allow learners to access new locations or interact with previously inaccessible content.

Examples:

* Climbing Boots unlock mountain paths
* A Compass reveals hidden routes
* A Knowledge Lens reveals invisible objects
* A Toolkit allows interaction with technical mechanisms

Tools create meaningful exploration progression.

---

# Multi-Function Tool Belt

Tools may be displayed in a dedicated tool belt interface.

The tool belt represents the learner's growing capabilities.

Some tools may be assembled from multiple discovered components.

Example:

```text
Engineering Toolkit

Components:
✓ Scanner Module
✓ Analyzer Module
✗ Diagnostic Module
```

When all components are discovered, the completed tool becomes available.

This creates long-term exploration goals.

---

# Hidden Content

The world may contain hidden or invisible elements.

Examples:

* Hidden portals
* Invisible doors
* Secret passages
* Bonus learning areas
* Optional challenges
* Easter eggs

Some hidden elements become visible only when specific tools are acquired.

Example:

```text
Knowledge Lens

Effect:
Reveal hidden objects within the world.
```

This encourages curiosity and rewards exploration.

---

# World Interaction

Items should not only exist inside the inventory.

The world itself should react to them.

Examples:

* Doors accept specific key stones
* Machines require tools
* NPCs request artifacts
* Portals require activation objects
* Hidden routes become accessible

This makes the world feel alive and responsive.

---

# Competency Integration

Items and tools may be connected to competencies.

Example:

```text
Climbing Boots

Unlocked by:
Mountain Navigation Competency
```

or

```text
ECE Keystone

Awarded when:
ECE Architecture Competency reaches Advanced level.
```

This ensures that world progression remains connected to learning progression.

---

# Core Principle

Items should not exist to increase a score.

Items should exist to expand possibilities.

The learner should feel:

"I can do something new now."

rather than:

"My number became bigger."
